## 2026-07-03

- cowswap-prompt-rework: candles default in price chart; large-mint prompt now recommends CoW Swap (both tabs, external link) with new `impact` (>1% truePriceImpact) and `capacity` (Ondo per-transaction cap from `dtf/ondo`) variants; `useOndoLimits` lifted to `src/hooks/`; new domain page [[zapper-prompt]].
- Lesson: `lingui extract` had not run since several features landed — the catalogs were stale enough that a routine extract swept in ~70 unrelated msgids. Run extract at every stage closeout that adds copy, or the next stage inherits the noise (this one filled its own 14 strings + the 4 chart-toggle labels the candles default made visible, and backlogged the rest).
- Lesson (review value): the Dark/Light pair caught a real latch hole (impact signal below the $100 applicability floor → mobile dialog re-pops after dismissal) and a product regression risk (empty candles endpoint → blank default chart, now falls back to the line body). Both were invisible to the gate; both were cheap to fix in-stage.
- cowswap-prompt-rework follow-ups (user testing): overview modal zapper now shows the prompt (`hideLargeMintPrompt` removed — flag pre-dated the rework); reducer got an escalation-only rule after real-wallet testing showed the card flickering between "High price impact" and "Large Order" as racing/refetching quotes disagreed around the 1% threshold. Latched variants never downgrade until a clean reset; 4 new reducer tests pin it.
- capacity warning: dropped the off-hours suppression after user testing ($5M input during the July-4 market closure showed "No route found" instead of "Order too large"). The signal is input-derived so it now fires pre-quote regardless of market state; closed-session caps are labeled "Regular" (the API mirrors regular caps when closed). Reverses the original "not during Off-Hours" spec line — explicit user instruction.
- Ondo off-hours rule (PM): CoW-CTA variants (large/impact/error) suppressed on Ondo DTFs while `market.isOpen === false` — off-hours secondary liquidity is stale, don't push users there. Note the inversion vs the same morning's change: capacity fires off-hours (informational, no CTA), CoW push does not. `!isOndoOffHours` also gates `isApplicable` so a latched CoW card clears when the market closes.
- referral tracking: `?referral=` capture (last-touch, no expiry) + `referral_landed` event + wallet↔code POST on connect; reserve-api gets `referral_links` table + origin-guarded `POST /referrals/link` (branch feature/referral-links there). New domain page [[referral]].
- Lesson (review value): Dark Claude caught that a bare `localStorage.setItem` inside the mixpanel-init effect would crash the app for Safari Private Mode visitors — the exact audience referral links target; all storage access now goes through safeGet/safeSet. Plan-stage review also killed two subgraph-API assumptions (index/yield query asymmetry, TRANSFER-in counted as mint) before any code was written — the feature then shrank to attribution-only and none of that code was needed.
- Fastify ordering gotcha: schema validation runs BEFORE preHandler, so a "guard" registered as preHandler does not shield validation; header-only guards belong on onRequest.
- referral follow-up (user-directed): reserve-api branch re-based onto origin/staging (deploy flow), localStorage helpers kept over atomWithStorage (no reactive consumer; dynamic flag keys don't fit atoms). /code-review high (workflow, 15 agents) then found what two Dark/Light passes missed — all three real bugs shared one root cause: captureReferral lived inside React's effect tree, where child effects (AtomUpdater link, Redirects navigate) run first. Fix was structural, not choreographic: capture to storage at module load in index.tsx, mixpanel half stays post-init. Lesson: when several findings share a root cause, fix the structure once instead of patching each ordering.
