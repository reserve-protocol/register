# Internationalization (i18n)

**Last Updated:** 2026-07-13

Register is localized with [Lingui v5](https://lingui.dev). Supported locales: **English (en),
Spanish (es), Korean (ko), Simplified Chinese (zh)**, plus a dev-only **pseudo** locale for
finding unwrapped strings. English is the default; an explicit choice is persisted to
`localStorage['register.locale']`.

## The one rule: reactive vs non-reactive

Locale switches live, with no page reload. For that to work, every translated string must read
from the **reactive** macros. The classic bug is a string that "doesn't update when you switch
languages" вҖ” it almost always means a non-reactive global `` t`...` `` slipped through.

| String lives inвҖҰ | Use | Import from |
|---|---|---|
| JSX text | `<Trans>` (and `<Plural>` for counts) | `@lingui/react/macro` |
| component body / event handler | `useLingui().t` | `@lingui/react/macro` |
| module scope (const arrays, option lists, zod messages, table headers) | `msg\`вҖҰ\`` вҶ’ render with reactive `t(descriptor)` | `@lingui/core/macro` |

**Never** use the global `` t`...` `` from `@lingui/macro` for runtime strings вҖ” it freezes to
the locale active at call time and won't re-render on switch.

### Canonical shapes

```tsx
// (a) JSX text
import { Trans } from '@lingui/react/macro'
<Trans>Connect wallet</Trans>

// (b) string computed in a component / handler
import { useLingui } from '@lingui/react/macro'
function Btn() {
  const { t } = useLingui()
  return <button title={t`Refresh`}>вҖҰ</button>
}

// (c) module-scope label array вҖ” msg now, reactive t() at render
import { msg } from '@lingui/core/macro'
import { useLingui } from '@lingui/react/macro'
import type { MessageDescriptor } from '@lingui/core'

const NAV: { id: string; label: MessageDescriptor }[] = [
  { id: 'overview', label: msg`Overview` },
]
function Nav() {
  const { t } = useLingui()
  return <>{NAV.map((i) => <span key={i.id}>{t(i.label)}</span>)}</>
}

// (d) zod schema that follows locale вҖ” build inside useMemo([t])
const M = { required: msg`Required` }
function useSchema() {
  const { t } = useLingui()
  return useMemo(() => z.object({ name: z.string().min(1, t(M.required)) }), [t])
}

// (e) toast in a handler
const T = { ok: msg`Auction launched successfully` }
function LaunchBtn() {
  const { t } = useLingui()
  return <button onClick={() => toast.success(t(T.ok))}>вҖҰ</button>
}
```

### Retrofitting legacy code

Older areas (mostly `src/views/yield-dtf/`) use the non-reactive global `t` from
`@lingui/macro`. When you touch one, migrate it:

- `import { t } from '@lingui/macro'` + render/handler `` t`...` `` вҶ’ `const { t } = useLingui()`
  (from `@lingui/react/macro`).
- module-scope `` t`...` `` вҶ’ `msg\`...\`` (from `@lingui/core/macro`), translated with reactive
  `t(descriptor)` at render.
- `<Trans>` usages need no change вҖ” they're already reactive via `I18nProvider`.

## Do NOT translate

Leave these as plain strings (they are not user-facing prose):

- token symbols / names, contract addresses (`0xвҖҰ`)
- chain names/ids (`ethereum`, `base`, `bnb`), enum/state literals (`buy`, `sell`, `idle`)
- `className` values, react-query keys, GraphQL field names
- route constants (`ROUTES.*`), URLs, time-range codes (`24h`, `7d`)

## Workflow

After wrapping strings in an area:

```bash
pnpm extract        # update src/locales/*.po with new/changed messages
```

Commit the code **and** the updated `.po` catalogs. `en.po` gets the source text.

**Adding or changing a user-facing string includes translating it вҖ” in the same change.**
Run `pnpm extract`, then fill the new `es/ko/zh` msgstr entries yourself (the agent generates
all translations; the glossary below keeps them consistent). Never leave empty msgstr behind.

Catalogs are loaded at runtime from `.po` by `@lingui/vite-plugin` (no `lingui compile` step вҖ”
the plugin compiles on import). Note for automated checks: the locale is stored JSON-encoded
(jotai `atomWithStorage`) вҖ” `localStorage.setItem('register.locale', JSON.stringify('es'))`.

`pnpm translations` runs `lingui extract --clean`, which additionally drops obsolete messages no
longer found in source вҖ” use it periodically to keep catalogs tidy.

### Verifying with the pseudo locale

The pseudo locale mangles every wrapped string (e.g. `Overview` вҶ’ `[Г”vГЁrvГӯГ«w]`), so anything
still in plain English is unwrapped. It's the primary tool for catching missed strings.

```js
// in the browser console, then reload
localStorage.setItem('register.locale', 'pseudo')
```

Pseudo is only selectable in dev (`import.meta.env.DEV`); in production the value is validated
and falls back to English. There is no language-switcher UI yet вҖ” set the locale via
`localStorage['register.locale']` (`en` / `es` / `ko` / `zh` / `pseudo`) and reload.

## Adding a locale

1. Add the two-letter code to `locales` in `lingui.config.ts`.
2. Add it to `SupportedLocale` and `SUPPORTED_LOCALES` in `src/i18n.tsx`.
3. Run `pnpm extract` to create `src/locales/<code>.po`, then fill its msgstr entries
   (glossary above).
