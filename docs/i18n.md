# Internationalization (i18n)

**Last Updated:** 2026-05-28

Register is localized with [Lingui v5](https://lingui.dev). Supported locales: **English (en),
Spanish (es), Korean (ko), Simplified Chinese (zh)**, plus a dev-only **pseudo** locale for
finding unwrapped strings. English is the default; an explicit choice is persisted to
`localStorage['register.locale']`.

## The one rule: reactive vs non-reactive

Locale switches live, with no page reload. For that to work, every translated string must read
from the **reactive** macros. The classic bug is a string that "doesn't update when you switch
languages" — it almost always means a non-reactive global `` t`...` `` slipped through.

| String lives in… | Use | Import from |
|---|---|---|
| JSX text | `<Trans>` (and `<Plural>` for counts) | `@lingui/react/macro` |
| component body / event handler | `useLingui().t` | `@lingui/react/macro` |
| module scope (const arrays, option lists, zod messages, table headers) | `msg\`…\`` → render with reactive `t(descriptor)` | `@lingui/core/macro` |

**Never** use the global `` t`...` `` from `@lingui/macro` for runtime strings — it freezes to
the locale active at call time and won't re-render on switch.

### Canonical shapes

```tsx
// (a) JSX text
import { Trans } from '@lingui/react/macro'
<Trans>Connect wallet</Trans>

// (b) string computed in a component / handler
import { useLingui } from '@lingui/react/macro'
function Btn() {
  const { t } = useLingui()
  return <button title={t`Refresh`}>…</button>
}

// (c) module-scope label array — msg now, reactive t() at render
import { msg } from '@lingui/core/macro'
import { useLingui } from '@lingui/react/macro'
import type { MessageDescriptor } from '@lingui/core'

const NAV: { id: string; label: MessageDescriptor }[] = [
  { id: 'overview', label: msg`Overview` },
]
function Nav() {
  const { t } = useLingui()
  return <>{NAV.map((i) => <span key={i.id}>{t(i.label)}</span>)}</>
}

// (d) zod schema that follows locale — build inside useMemo([t])
const M = { required: msg`Required` }
function useSchema() {
  const { t } = useLingui()
  return useMemo(() => z.object({ name: z.string().min(1, t(M.required)) }), [t])
}

// (e) toast in a handler
const T = { ok: msg`Auction launched successfully` }
function LaunchBtn() {
  const { t } = useLingui()
  return <button onClick={() => toast.success(t(T.ok))}>…</button>
}
```

### Retrofitting legacy code

Older areas (mostly `src/views/yield-dtf/`) use the non-reactive global `t` from
`@lingui/macro`. When you touch one, migrate it:

- `import { t } from '@lingui/macro'` + render/handler `` t`...` `` → `const { t } = useLingui()`
  (from `@lingui/react/macro`).
- module-scope `` t`...` `` → `msg\`...\`` (from `@lingui/core/macro`), translated with reactive
  `t(descriptor)` at render.
- `<Trans>` usages need no change — they're already reactive via `I18nProvider`.

## Do NOT translate

Leave these as plain strings (they are not user-facing prose):

- token symbols / names, contract addresses (`0x…`)
- chain names/ids (`ethereum`, `base`, `bnb`), enum/state literals (`buy`, `sell`, `idle`)
- `className` values, react-query keys, GraphQL field names
- route constants (`ROUTES.*`), URLs, time-range codes (`24h`, `7d`)

## Workflow

After wrapping strings in an area:

```bash
pnpm extract        # update src/locales/*.po with new/changed messages
```

Commit the code **and** the updated `.po` catalogs. `en.po` gets the source text.

**Adding or changing a user-facing string includes translating it — in the same change.**
Run `pnpm extract`, then fill the new `es/ko/zh` msgstr entries yourself (the agent generates
all translations; the glossary below keeps them consistent). Never leave empty msgstr behind.
Crowdin remains the review layer: if translators later refine an entry, their version wins via
the scheduled `chore(i18n)` PR.

Catalogs are loaded at runtime from `.po` by `@lingui/vite-plugin` (no `lingui compile` step —
the plugin compiles on import). Note for automated checks: the locale is stored JSON-encoded
(jotai `atomWithStorage`) — `localStorage.setItem('register.locale', JSON.stringify('es'))`.

`pnpm translations` runs `lingui extract --clean`, which additionally drops obsolete messages no
longer found in source — use it periodically (and before a Crowdin source sync) to keep catalogs
tidy.

### Glossary (agent-generated translations)

Match the existing catalogs' voice: **es uses informal "tú"**, ko uses plain polite forms
(-합니다/-하세요 in prose, noun phrases for labels), zh uses 你 and full-width punctuation (：！).
Product verbs/nouns that stay in English in **every** locale: **Mint, Redeem, Zap, Stake /
Unstake / Staker, Wallet (capitalized), vote-lock, DTF, TVL, tokens/symbols/tickers**.

| en | es | ko | zh |
|---|---|---|---|
| balance | balance | 잔액 | 余额 |
| portfolio | portafolio | 포트폴리오 | 投资组合 (nav item "Portfolio" = 我的资产) |
| collateral | colateral | 담보 | 抵押品 |
| rebalance | reequilibrio | 리밸런싱 | 再平衡 |
| governance | gobernanza | 거버넌스 | 治理 |
| proposal | propuesta | 제안 | 提案 |
| manual / automated | manual / automatizado | 수동 / 자동 | 手动 / 自动 |
| market(s) | mercado(s) | 마켓 (DEX venues) · 시장 (the market) | 市场 |
| fee | comisión | 수수료 | 费用 |
| basket | cesta | 바스켓 | 篮子 |

When unsure, grep the catalog for a nearby existing msgid and copy its terminology — the
catalogs are the source of truth, this table is the index.

### Verifying with the pseudo locale

The pseudo locale mangles every wrapped string (e.g. `Overview` → `[Ôvèrvíëw]`), so anything
still in plain English is unwrapped. It's the primary tool for catching missed strings.

```js
// in the browser console, then reload
localStorage.setItem('register.locale', 'pseudo')
```

Pseudo is only selectable in dev (`import.meta.env.DEV`); in production the value is validated
and falls back to English. There is no language-switcher UI yet — set the locale via
`localStorage['register.locale']` (`en` / `es` / `ko` / `zh` / `pseudo`) and reload.

## Adding a locale

1. Add the two-letter code to `locales` in `lingui.config.ts`.
2. Add it to `SupportedLocale` and `SUPPORTED_LOCALES` in `src/i18n.tsx`.
3. Run `pnpm extract` to create `src/locales/<code>.po`.
4. Add the target language in the Crowdin project (see below); confirm its download path
   resolves to `src/locales/<code>.po` (use `languages_mapping` in `crowdin.yml` if Crowdin's
   two-letter code differs, as we do for `zh-CN → zh`).

## Crowdin pipeline

Translation sync runs through Crowdin via **GitHub Actions** (not the Crowdin GitHub App). Two
workflows, mapped by `crowdin.yml` (source `en.po` → `%two_letters_code%.po`):

- **`crowdin-upload.yml`** — on push to `master` touching `src/locales/en.po` or `crowdin.yml`.
  `permissions: contents: read` (cannot modify the repo); only uploads the English source.
- **`crowdin-download.yml`** — on a 6-hour cron + `workflow_dispatch`. Pulls finished
  translations and **opens a PR** (`l10n_master` → `master`, title `chore(i18n): update
  translations`). It never merges; it should only ever touch `src/locales/*.po`.

This repo manages money and the team has been burned by malicious npm packages, so the pipeline
is deliberately hardened:

- All actions are **pinned to full commit SHAs** (not tags). `dependabot.yml` opens reviewable
  PRs to bump them. When updating a pin, re-resolve the tag to its commit SHA — never trust a
  tag alone.
- Checkout uses `persist-credentials: false`; workflows use `concurrency` groups.
- `@crowdin/cli` is **not** a project dependency — the action bundles it. For a rare one-off
  manual sync, use a pinned `pnpm dlx @crowdin/cli@<version> …` rather than installing it.

### Manual GitHub settings (apply once in the GitHub UI — code can't set these)

The "automation can't land translations on `master` without a human" guarantee rests on these,
**not** on the workflow YAML:

- **Branch protection on `master`:** require PR + approval; dismiss stale approvals on new
  commits; "Do not allow bypassing the above settings" ON; block force-pushes/deletions.
- **Settings → Actions → General:** default workflow token = **read-only**; **"Allow GitHub
  Actions to create and approve pull requests" = OFF** (so the download PR can't self-approve).
- **Secrets (Actions):** `CROWDIN_PROJECT_ID`, `CROWDIN_PERSONAL_TOKEN`.

### Verifying the pipeline

1. Push an `en.po` change to `master` → `crowdin-upload` runs (read-only) and the strings appear
   in Crowdin.
2. `workflow_dispatch` `crowdin-download` → it opens an `l10n_master` PR touching only
   `src/locales/*.po`. Confirm the file list before approving.
